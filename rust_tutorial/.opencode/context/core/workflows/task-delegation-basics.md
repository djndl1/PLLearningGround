# Rust Tutorial Task Delegation Workflow

## Overview
Simple task delegation for the Rust Tutorial project. Focus on basic help when needed.

## When to Delegate

### Simple Tasks (Do Yourself)
- Fix small bugs
- Write simple functions
- Add basic comments
- Run tests

### Complex Tasks (Get Help)
- Write complex code
- Create new features
- Write comprehensive tests
- Generate documentation

## Delegation Process

### Step 1: Check if you can do it
**Quick assessment:**
- Can I finish in <30 minutes?
- Do I understand the task?
- Do I have all needed info?

### Step 2: Get help if needed
**Ask for help when:**
- Task takes too long
- Code is complex
- Need specialized knowledge
- Want better examples

### Step 3: Use help effectively
**When getting help:**
- Be clear about what you need
- Tell them what you already tried
- Ask specific questions
- Show your current work

## Getting Help

### Ask for help with:
- Complex Rust code
- Advanced concepts
- Better examples
- Testing strategies
- Documentation

### What to tell them:
- What you're trying to do
- What you've already tried
- What you need help with
- Any errors you're seeing

## Simple Workflow

### If you can do it:
1. Start working
2. Ask questions if stuck
3. Finish and check

### If you need help:
1. Tell me what you need
2. Show what you've done
3. Ask specific questions
4. Get guidance
